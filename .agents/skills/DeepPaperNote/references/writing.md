# Evidence-First Writing

## 目录

- 写作目标
- Note plan
- 双层结构
- 论文类型适配
- 技术深度
- 质量与可读性复核

## 写作目标

把 synthesis bundle 当作证据输入，而不是可直接复制的提纲。最终笔记要重建论文的论证：研究问题、任务或对象、数据或材料、方法或推导、关键结果、证据边界、未证明内容和后续价值。

区分作者声称、直接测量或计算结果、模型依赖推断以及尚未排除的替代解释。只使用摘要时必须标记为临时或 degraded。

## Note plan

写正文前生成简短 JSON note plan，至少确定：

- paper_type 与 dominant_domain。
- must_cover 及其 evidence_id。
- 3 至 6 个关键数字和真正的比较。
- key_claims、章节权重和必要的三级标题。
- 是否需要公式、复杂度、训练/推理细节。
- 每个重要图表的阅读目的。

不要暴露长篇自由思维记录。

## 双层结构

每篇笔记保持单文件：

快速层通常包含 30 秒速览、关键结论、关键数字、适用边界、快速入口与页面导航，以及必要时的术语表。

深读层包含原文摘要翻译、创新点、研究问题、领域适配的方法或理论、主要结果与证据链、解释与替代解释、局限与未决问题、可复用结论、相关论文、我的笔记和引用。

原文摘要翻译只忠实翻译摘要，不混入后见判断。创新点通常列出 3 至 5 个论文特定贡献并说明价值。核心信息只放元数据。

## 论文类型适配

- experimental_physics：按系统/控制、直接观测、分析/校准、物理推断、替代解释组织。
- theoretical_physics：强调假设、模型、推导、预测与适用边界。
- materials_fabrication：强调材料、工艺步骤、加工窗口、表征和器件结果。
- ai_method：强调问题设置、机制、训练与推理、基线、消融和成本。
- benchmark：强调数据构建、任务拆分、偏差和真正测量的能力。
- clinical：强调样本、变量、监督、分析、临床意义和泛化边界。
- humanities：强调材料、理论框架、论证路径、概念贡献和解释限制。
- survey：强调范围、分类框架、共识、争议和缺口。
- generic：只在无法可靠分类时使用。

不要默认套用 AI 输入—操作—输出模板。

## 技术深度

技术论文必须解释信息或因果链，而不是列模块名。方法、系统或框架论文通常在方法章节加入 3 至 4 步机制流程，每步说明输入、操作和输出。

保留真正影响理解的目标函数、概率分解、更新规则、复杂度或标度公式。使用 Obsidian/MathJax 可直接渲染的行内或块级 LaTeX；每个公式后解释它在实现或物理上对应什么。

结果只选最有解释力的数字与比较，同时写出限制结论的结果。若没有失败或不稳定消融，明确说明论文没有清楚报告，而不是编造。

## 质量与可读性复核

质量复核至少检查证据忠实度、完整性、领域适配、技术深度和可追溯性，每项低于 4/5 都不能正常发布。至少核对三个核心结论与 evidence_id。

lint 通过后完整重读中文正文，修正生硬翻译、普通英文残留、PDF 断行和为通过规则而写的空话。模型名、数据集名、指标、方法名、数学符号、代码标识和原始 Fig./Table 编号可以保留英文。

可读性复核不能引入新事实或改变关键数字。复核若改动正文，必须重跑 lint，并重新绑定质量与可读性报告。
