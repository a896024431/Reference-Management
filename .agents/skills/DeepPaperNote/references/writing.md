# Evidence-First Writing

## 写作目标

把 synthesis bundle 当作证据输入，重建论文的研究问题、对象、方法或推导、关键结果、解释边界与后续价值。区分作者主张、直接测量或计算、模型依赖推断和未排除的替代解释。

只有完整全文与关键证据强制检查通过后才写正式笔记；摘要或关键证据不足时停止并报告，不生成降级正文。

## Note plan

note plan 使用唯一结构。顶层八个字段见 `workflow.md`，条目结构为：

- `must_cover[]`：`topic` 与 `evidence_ids`
- `key_claims[]`：`claim` 与 `evidence_ids`
- `key_numbers[]`：`number` 与 `evidence_ids`
- `real_comparisons[]`：`comparison` 与 `evidence_ids`
- `section_plan[]`：`section` 与 `evidence_ids`

`must_cover`、`key_claims` 和 `section_plan` 不得为空。确实没有关键数字或真实比较时，对应数组可以为空。顶层 `evidence_ids` 是各条目所用证据编号的无重复索引，不接受字符串条目或缺少证据关联的宽松变体。

## 双层结构

快速层通常包含 30 秒速览、至少三条关键结论、关键数字、适用边界、快速入口，以及必要时的术语表。每条关键结论自身必须带主文或补充材料页码；快速入口中的页内链接必须精确命中当前标题。

深读层包含摘要翻译、创新点、研究问题、领域适配的方法或理论、主要结果与证据链、解释与替代解释、局限与未决问题、可复用结论、相关论文、我的笔记和引用。

原文摘要翻译只忠实翻译摘要。核心信息只放元数据。英文题名最多出现一次；正文只有一个 H1。

## 论文类型适配

- `experimental_physics`：系统/控制、直接观测、分析/校准、物理推断、替代解释。
- `theoretical_physics`：假设、模型、推导、预测与适用边界。
- `materials_fabrication`：材料、工艺步骤、加工窗口、表征和器件结果。
- `ai_method`：问题设置、机制、训练与推理、基线、消融和成本。
- `benchmark`：数据构建、任务拆分、偏差和实际测量能力。
- `clinical`：样本、变量、监督、分析、临床意义和泛化边界。
- `humanities`：材料、理论框架、论证路径、概念贡献和解释限制。
- `survey`：范围、分类框架、共识、争议和缺口。
- `generic`：只在无法可靠分类时使用。

不要默认套用 AI 输入—操作—输出模板。

## 技术深度与数学

解释信息链或因果链，不把模块、术语或数字堆积当作分析。保留真正影响理解的公式，并在相邻正文解释符号、适用条件和它回答的问题。

LaTeX 命令只能出现在 `$...$`、`$$...$$`、`\(...\)` 或 `\[...\]` 中；代码标识使用反引号，数学公式不能用代码格式。

## 第二读校对

第二读同时核对忠实度、完整性、领域表达、清晰度、中文自然度、导航和可追溯性。它至少引用当前正文中三个不同的段落；每条引文必须位于所声明的实际 Markdown 标题下，并逐条关联 evidence ID。

第二读必须由不同于作者的另一代理或人工完成，来源记录为 `subagent` 或 `human`。每项低于 4/5、存在遗留问题，或正文修改后原校对不再对应当前版本，都不能发布。任何正文修改都要重新 lint 并重新校对。

第二读应在不继承作者对话历史的独立上下文中完成；它必须直接读取当前 run 的笔记和本地证据，不能把作者的解释或 note plan 当作证据替代。
